## Flex

Once UI layouts are built using `Flex` and `Grid`, prop-based abstractions over CSS flexbox and grid. These primitives are the foundation of the system, enabling layout logic through composition rather than CSS.

<Flex
  marginTop="16"
  marginBottom="24"
  fillWidth
  gap="12"
  radius="l"
  padding="l"
  border="neutral-medium"
  background="surface"
>
  {Array(6)
    .fill(null)
    .map((_, index) => (
      <Background 
        lines={{display: true, color: "neutral-alpha-weak"}}
        key={index}
        shadow="m"
        fillWidth
        radius="m"
        height={8}
        border="neutral-alpha-medium"
      />
    ))}
</Flex>

### Syntax

We expose two convenience wrappers: `Row` and `Column`. These are semantically expressive `Flex` elements with pre-set directions. Use them instead of the `Flex` for better semantics and code quality.

### Alignment

Rather than wrestling with `justifyContent` and `alignItems`, we use semantic `horizontal` and `vertical` props that auto-adjust based on direction.

<CodeBlock
  marginTop="8"
  marginBottom="16"
  codePreview={
    <Column fillWidth gap="16">
      <Background
        lines={{display: true, color: "neutral-alpha-weak"}}
        fillWidth minHeight={10} padding="16" radius="m" border="neutral-alpha-medium">
        <Button size="s" data-border="rounded">Continue</Button>
      </Background>
      <Background
        lines={{display: true, color: "neutral-alpha-weak"}}
        fillWidth center minHeight={10} padding="16" radius="m" border="neutral-alpha-medium">
        <Button size="s" data-border="rounded">Continue</Button>
      </Background>
      <Background
        lines={{display: true, color: "neutral-alpha-weak"}}
        fillWidth horizontal="end" vertical="end" minHeight={10} padding="16" radius="m" border="neutral-alpha-medium">
        <Button size="s" data-border="rounded">Continue</Button>
      </Background>
    </Column>
  }
  codeInstances={[
    {
      code:
`<Column fillWidth minHeight={10}>
    <Button>Continue</Button>
</Column>
<Column fillWidth center minHeight={10}>
    <Button>Continue</Button>
</Column>
<Column fillWidth horizontal="end" vertical="end" minHeight={10}>
    <Button>Continue</Button>
</Column>
`,
      language: "tsx",
      label: "Alignment"
    }
  ]}
/>

The `start` is the default value for both alignments, so we don't declare them. The `center` prop is shorthand for both `horizontal="center"` and `vertical="center"`.

We can also use `space-around` and `space-between` values for the alignment props:

      </Background>
      <Background

        <IconButton icon="chevronLeft" variant="secondary" data-border="rounded"/>
        <Button size="s" data-border="rounded">Continue</Button>

</Column>

### Size

Use the `fillWidth`, `fillHeight`, or shorthand `fill` to fill up the remaining space. These properties also apply `minWidth` and `minHeight` of `0` if not specified.

Use `fit`, `fitWidth`, `fitHeight` to apply `fit-content`.

<CodeBlock
  marginTop="8"
  marginBottom="16"
  codePreview={
    <Column fillWidth horizontal="center" gap="16">
      <Background
        border="neutral-alpha-medium" radius="m" padding="16"
        lines={{display: true, color: "neutral-alpha-weak"}}
        fit>
        <Button size="s" data-border="rounded">Continue</Button>
      </Background>
      <Background
        horizontal="center" border="neutral-alpha-medium" radius="m" padding="16"
        lines={{display: true, color: "neutral-alpha-weak"}}
        fillWidth fitHeight>
        <Button size="s" data-border="rounded">Continue</Button>
      </Background>
    </Column>
  }
  codeInstances={[
    {
      code:
`<Column fit background="overlay">
    <Button>Continue</Button>
</Column>
<Column fillWidth fitHeight horizontal="center" background="overlay">
    <Button>Continue</Button>
</Column>
`,
      language: "tsx",
      label: "Fill"
    }
  ]}
/>

You can also use `maxWidth`, `minHeight`, and `aspectRatio`. `maxWidth` will also apply `fillWidth`.

<CodeBlock
  marginTop="8"
  marginBottom="16"
  codePreview={
    <Column fitHeight fillWidth horizontal="center" gap="16" align="center" onBackground="neutral-weak" textVariant="label-default-s">
      <Background maxWidth={20}
        center border="neutral-alpha-medium" radius="m" padding="16"
        lines={{display: true, color: "neutral-alpha-weak"}}>
        maxWidth: 20
      </Background>
      <Background
        center border="neutral-alpha-medium" radius="m" padding="16"
        lines={{display: true, color: "neutral-alpha-weak"}}
        aspectRatio="16/9">
        aspectRatio: 16 / 9
      </Background>
    </Column>
  }
  codeInstances={[
    {
      code:
`<Column maxWidth={20}>
    maxWidth={20}
</Column>
<Column fillWidth aspecRatio="16/9>
    aspectRatio="16 / 9"
</Column>
`,
      language: "tsx",
      label: "Fill"
    }
  ]}
/>

### Spacing

Use utility props to apply spacing consistently, such as the `padding`, `paddingX`, `marginTop`, etc. Values can be [spacing tokens](/once-ui/basics/spacing) or numbers (converted to `rem`). Use the `gap` to set the internal spacing between children.

### Border

Apply borders using:

- `border="neutral-medium"` or `brand-alpha-weak`
- `borderWidth`, `borderStyle`
- `border="transparent"` to reserve space for layout consistency

Use `borderRadius` or individual props like `topLeftRadius`.

## Interactivity

- `cursor="interactive"` uses a design tokenized pointer
- `shadow="m"` applies a predefined elevation
- `zIndex` is a scale from 0–10
- `transition="micro-medium"` sets duration presets

## Overflow & Opacity

- `overflowX`, `overflowY`, or `overflow`
- `opacity` from `0` to `100` in steps of `10`


## Responsiveness

## Visibility

Render conditionally with:

- `hide="s"` or `hide="m"`
- `show="l"` etc.
- `dark` or `light` props to limit rendering by theme


## Typography

Use the `textVariant` prop to apply text styles directly:

- `textVariant="body-default-s"`
- Align text with `align="center"`


## Composition

All layout components:

- Use `forwardRef`
- Expose `className`, `style`, and full layout props
- Spread props on the outermost `Flex` wrapper

That’s the Once UI layout model — composable, prop-based, responsive by default, and production-oriented.

You now have a low-code Figma-to-code framework — not just a design system, but a layout language.
