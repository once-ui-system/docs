## Flex

Once UI layouts are built using `Flex` and `Grid`, prop-based abstractions over CSS flexbox and grid. These primitives are the foundation of the system, enabling layout logic through composition rather than CSS.

### Syntax Basics

We expose two convenience wrappers: `Row` and `Column`. These are semantically expressive `Flex` elements with pre-set directions. Use them instead of the `Flex` for better semantics and code quality.

### Alignment

Rather than wrestling with `justifyContent` and `alignItems`, we use semantic `horizontal` and `vertical` props that auto-adjust based on direction.

<CodeBlock
  codeInstances={[
    {
      code: `
<Column horizontal="center" vertical="end" fillHeight>
  <Button variant="solid" color="brand">Continue</Button>
</Column>
`,
      language: "tsx",
      label: "Semantic alignment"
    }
  ]}
/>

The `center` prop is shorthand for both `horizontal="center"` and `vertical="center"`.

### Spacing

Use utility props to apply spacing consistently:

- `padding`, `paddingX`, `marginTop`, etc.
- Values can be spacing tokens (`"6"`, `"12"`) or numbers (converted to `rem`)
- `gap` sets the internal spacing between children

<CodeBlock
  codeInstances={[
    {
      code: `
<Column padding="8" gap="4">
  <Text>Line 1</Text>
  <Text>Line 2</Text>
</Column>
`,
      language: "tsx",
      label: "Padding and gap"
    }
  ]}
/>


### Size

Use the `fillWidth`, `fillHeight`, or shorthand `fill` to apply `100%` dimensions.

<CodeBlock
  codeInstances={[
    {
      code: `
<Flex fill background="overlay">
  <Text variant="heading-strong-l">Fullscreen Section</Text>
</Flex>
`,
      language: "tsx",
      label: "Fill size"
    }
  ]}
/>

Use `fit`, `fitWidth`, `fitHeight` to apply `fit-content`.

You can also use `maxWidth`, `minHeight`, and `aspectRatio`.


### Border

Apply borders using:

- `border="neutral-medium"` or `brand-alpha-weak`
- `borderWidth`, `borderStyle`
- `border="transparent"` to reserve space for layout consistency

Use `borderRadius` or individual props like `topLeftRadius`.

<CodeBlock
  codeInstances={[
    {
      code: `
<Flex
  border="neutral-medium"
  borderRadius="xl"
  padding="6"
>
  <Text>Bordered element</Text>
</Flex>
`,
      language: "tsx",
      label: "Border and radius"
    }
  ]}
/>


## Interactivity

- `cursor="interactive"` uses a design tokenized pointer
- `shadow="m"` applies a predefined elevation
- `zIndex` is a scale from 0–10
- `transition="micro-medium"` sets duration presets

## Overflow & Opacity

- `overflowX`, `overflowY`, or `overflow`
- `opacity` from `0` to `100` in steps of `10`


## Responsiveness

Override layout flow with `mobileDirection` or `tabletDirection`.

<CodeBlock
  codeInstances={[
    {
      code: `
<Column mobileDirection="row" gap="4">
  <Avatar fallback="A" />
  <Text>Mobile switches to horizontal</Text>
</Column>
`,
      language: "tsx",
      label: "Responsive direction"
    }
  ]}
/>


## Visibility

Render conditionally with:

- `hide="s"` or `hide="m"`
- `show="l"` etc.
- `dark` or `light` props to limit rendering by theme


## Typography

Use the `textVariant` prop to apply text styles directly:

- `textVariant="body-default-s"`
- Align text with `align="center"`


## Composition

All layout components:

- Use `forwardRef`
- Expose `className`, `style`, and full layout props
- Spread props on the outermost `Flex` wrapper

That’s the Once UI layout model — composable, prop-based, responsive by default, and production-oriented.

You now have a low-code Figma-to-code framework — not just a design system, but a layout language.
